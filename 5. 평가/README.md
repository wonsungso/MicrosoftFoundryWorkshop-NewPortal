# 5. 평가

생성형 AI 애플리케이션은 모델 자체보다도 **실제 데이터와 프롬프트, 에이전트 설정을 포함한 전체 시스템**의 품질을 평가해야 합니다.

[Microsoft Foundry 포털](https://ai.azure.com/)에서는 모델, 에이전트, 데이터셋을 대상으로 평가를 실행하고 결과를 비교할 수 있습니다.

---

## 애플리케이션 성능 평가 및 비교

이번 실습에서는 `FundRecommendationAgent`의 추천 결과를 데이터셋 기반으로 평가합니다.

금융상품 추천은 정답이 하나로 고정되지 않는 경우가 많기 때문에, 다음과 같은 품질 축을 함께 보는 것이 중요합니다.

- **관련성**: 질문에 맞는 답인지
- **근거성**: 제공된 컨텍스트와 추천 근거가 연결되는지
- **일관성**: 추천 형식과 설명 구조가 안정적인지

## 평가 실행

1. [Microsoft Foundry 포털](https://ai.azure.com/)에서 왼쪽 메뉴의 `평가`로 이동합니다.
2. `새 평가 만들기`를 클릭합니다.
3. 평가 대상에서 `데이터 세트`를 선택합니다.
4. `새 데이터 세트 업로드`를 클릭합니다.
5. [fund_products_eval_sample.jsonl](./../assets/fund_products_eval_sample.jsonl) 파일을 업로드합니다.
6. 업로드가 끝나면 데이터 미리보기를 확인하고 다음 단계로 이동합니다.

## 평가자 선택

[Microsoft Foundry 포털](https://ai.azure.com/)에서는 품질 평가자와 안전 평가자를 함께 선택할 수 있습니다. 이번 실습에서는 품질 평가 위주로 진행합니다.

1. 평가자 선택 단계에서 `Likert 배율 계산기`를 우선 선택합니다.
2. 필요하면 `모델 채점자`나 다른 품질 평가자를 추가합니다.
3. AI judge 모델이 필요한 평가자라면 프로젝트에 배포된 `gpt-4o` 또는 `gpt-4o-mini` 계열 모델을 선택합니다.
4. 필드 매핑 화면이 나오면 다음과 같이 연결합니다.

- 쿼리: `query`
- 응답: `response`
- 컨텍스트: `context`

5. 평가 이름은 `FundRec_eval_v1`처럼 입력합니다.
6. 마지막에 `제출`을 클릭합니다.

![image.png](./images/image_evaluation_submit.png)

> 참고
> `Likert 배율 계산기`는 관련성, 근거성, 일관성 같은 항목을 1점부터 5점 척도로 보기 좋아서 워크샵 실습용으로 적합합니다.

## 결과 확인

평가가 완료되면 상단 탭에서 결과를 확인합니다.

- **선고 탭**: 평가 실행 상태, 메트릭 대시보드, 평가자별 통과율 요약
- **데이터 탭**: 행 단위 상세 결과 (`Detailed metrics result`). 각 샘플의 점수, Pass/Fail, 컨텍스트·쿼리·응답 내용을 비교할 수 있습니다.

평가 이름에 `FundRec_Likert_v1`처럼 버전 정보를 포함하면 이후에 프롬프트를 수정한 뒤 새 평가결과와 비교하기 쉽습니다.

## 해석 포인트

특히 아래 상황을 중점적으로 봅니다.

- 관련성은 높은데 근거성이 낮은 경우
    - 추천은 그럴듯하지만 업로드된 펀드 데이터와 연결이 약할 수 있습니다.
- 근거성은 높은데 응답 품질이 낮은 경우
    - 검색은 되었지만 설명 구조나 추천 요약이 부족할 수 있습니다.
- 특정 조건의 질문에서만 점수가 낮은 경우
    - Connected agent 호출 방식이나 프롬프트 설계가 불안정할 수 있습니다.

이 결과를 바탕으로 지침, 연결된 에이전트 설명, 지식 구성을 조정한 뒤 다시 평가를 실행하면 개선 효과를 비교할 수 있습니다.
